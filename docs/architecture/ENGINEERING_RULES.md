# DP Chain Engineering Rules

## Rule 1
Never modify upstream code unless absolutely necessary.

## Rule 2
Understand first. Modify second. Build third. Optimize last.

## Rule 3
Every major architectural decision must be documented before implementation.

## Rule 4
A working blockchain is more valuable than an unfinished perfect blockchain.

## Rule 5
Every release must leave the project in a usable state.

## Rule 6
Security is always more important than convenience.

## Rule 7
Performance improvements must never compromise correctness.

## Rule 8
Automation over manual operations whenever possible.

## Rule 9
Everything should eventually be manageable through a web interface.

## Rule 10
Keep DP Chain easier to deploy, operate and maintain than competing Layer 1 blockchains.

## Rule 11
Never sacrifice long-term maintainability for short-term speed.

## Rule 12
Every feature should have a clear purpose.
